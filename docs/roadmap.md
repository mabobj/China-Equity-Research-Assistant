# 路线图

本文回答三个问题：

1. 项目长期要走向哪里
2. 当前已经完成到什么位置
3. 后续应该按什么顺序推进

## 1. 北极星目标

项目长期方向已经明确收敛为：

**以公开信息为基础的 A 股研究、预测、验证、组合与监控系统**

长期能力链条是：

`数据底座 -> 点时特征 -> 标签 -> 回测 -> 预测 -> 评估 -> 交易反馈 -> 因子验证 -> 组合与监控`

## 2. 当前状态

当前项目已经完成的基础层包括：

- 单票工作台
- 选股 workflow 与深筛 workflow
- 交易记录与复盘最小闭环
- `bars / financial_summary / announcements` 清洗层
- 日级数据产品与 freshness 体系
- `prediction / backtest / evaluation` 最小底座
- 数据血缘与版本追踪主链
- 初筛因子体系第一轮工程化落地

也就是说，当前已经具备：

- 研究与决策工作台产品壳
- 预测与评估主线的第一版接入
- 因子化初筛链的第一版落地

## 3. 当前阶段判断

当前正确的推进方向不是继续横向堆更多页面，而是：

1. 让数据底座更适合长期因子与预测主线
2. 让预测与评估逐步成为可验证的核心增量
3. 让现有工作台在此基础上继续提升消费级体验

## 4. 后续阶段路线

### 阶段 A：数据底座继续收口

目标：

- provider 能力表清晰
- 清洗层扩展到长期需要的数据域
- 点时一致性继续工程化
- 复权、公司行为、交易状态口径继续收口

完成标准：

- 关键数据域都能说明来源、质量、版本与重建方式
- 特征与标签构建不再依赖临时拼接数据

### 阶段 B：点时特征与标签资产化

目标：

- 建立稳定的 point-in-time 特征资产
- 建立统一标签体系
- 明确 `feature_version / label_version`

完成标准：

- 任意 `symbol + as_of_date` 可重建特征
- 任意回测样本可追溯标签来源

### 阶段 C：回测与评估主线增强

目标：

- baseline 回测可复现
- walk-forward 评估更稳定
- 评估结果服务于模型版本治理

完成标准：

- 可以回答“某版本是否优于基线”
- 可以回答“某种候选排序是否真的有增量”

### 阶段 D：预测结果深度接回产品

目标：

- 单票页、选股页、deep-review 页统一消费预测结果
- 研究解释链与预测评估链形成并行对照

完成标准：

- 用户可以区分“研究结论”和“预测结论”
- workflow 结果能显示可追溯的排序依据与版本信息

### 阶段 E：交易反馈服务验证主线

目标：

- 交易与复盘不再只是记录工具
- 为规则评估、模型评估、因子验证提供真实反馈

完成标准：

- 能回答“哪些结论有效，哪些只是解释好看”
- 能回答“哪些质量门控合理，哪些偏保守”

### 阶段 F：因子发现、验证、组合与监控

目标：

- 从产品工作台升级到因子工程系统
- 建立验证、组合约束、衰减监控等长期能力

完成标准：

- 因子不再只停留在页面解释
- 因子可以进入验证、筛选、组合和持续监控体系

## 5. 当前明确不做的方向

在进入上述长期阶段前，以下内容仍不作为当前主线：

- 自动下单
- 券商接入
- 高频交易
- 未经人工确认的交易执行
- 把 LLM 自由文本直接当交易决策引擎

## 7. 下一步建议

如果按“符合长期方向、又不过度扩散”的原则，接下来最合理的顺序是：

1. 继续补齐与长期因子主线直接相关的数据底座缺口
2. 推进点时特征、标签、回测、评估资产化
3. 让初筛因子快照进入后续验证与复用链路
4. 再把预测结果更深地接回单票、选股、交易、复盘
