# 开发路线图

## 目标

本文档定义 A 股研究助手的阶段性开发计划与优先级。

总体原则：
- 先把架构打稳
- 先后端后前端
- 先数据与结构，再增强表达
- 先 MVP，再扩展
- 先研究系统，再考虑更强 agent 化

## 当前状态（2026-04-01）

已实质落地：
- 单票主链路：`review-report v2`、`debate-review`、`strategy plan`、`decision brief`
- 单票主入口：`workspace-bundle`（含模块状态、证据清单、freshness 摘要）
- 选股主链路：workflow 模式（含 run 记录、轮询查询、局部失败可见）
- 初筛第一阶段产品化：互斥运行、游标分批、17:00 窗口口径、批次台账与结果查询
- 日级数据产品复用：`review_report_daily`、`debate_review_daily`、`strategy_plan_daily` 等
- Data 清洗层 v0.1：`bars`、`financial_summary`、`announcements` 已接入主链路
- quality_status 消费已落地：`/research` 置信度修正 + `/screener` 分桶门控与质量提示
- 交易与复盘闭环 v0.1：`decision snapshot -> trade -> review` 已可用，含一致性校验与复盘草稿

当前主要技术债：
- Python 3.9 兼容约束下的测试/运行环境差异
- 少量体验型优化（非主链路稳定性风险）

## v2.1 推进方式（功能轨 + 体验轨）

自本版本起，项目进入“功能主线与体验主线并行”阶段，避免只堆模块导致可用性下降。

- 功能轨：继续推进预测与评估能力（特征、标签、回测、预测、评估、版本化）。
- 体验轨：重构单票/选股/交易/复盘页面信息架构，统一术语与操作路径。
- 约束：每个任务包必须同时包含代码、测试、文档；体验验收不通过，不进入下一功能包。

详见：
- [项目任务书 v2.1（产品化增强版）](taskbook-v2.1.md)

## 阶段划分

- Phase 0：项目初始化
- Phase 1：数据层 MVP
- Phase 2：技术分析 MVP
- Phase 3：单票研究 MVP
- Phase 4：选股器 MVP
- Phase 5：交易记录与复盘 MVP
- Phase 6：轻前端完善
- Phase 7：策略优化与学习增强

## Phase 0：项目初始化

### 目标
建立可开发、可迭代的项目骨架。

### 输出
- 仓库目录结构
- README
- AGENTS.md
- architecture.md
- roadmap.md
- 后端初始化
- 前端初始化
- 基础脚本
- `.env.example`
- `.gitignore`

### 完成标准
- 本地可启动后端
- 本地可启动前端
- 项目文档齐全
- 基础目录固定

## Phase 1：数据层 MVP

### 目标
建立第一版 A 股数据接入和标准化能力。

### 范围
- 接入 AKShare
- 接入 Baostock
- 接入公告源
- 基础新闻源
- 标准化与缓存

### 关键能力
- 获取股票基础信息
- 获取日线行情
- 获取公告列表
- 获取简要新闻列表
- 获取基础市场股票池

### 完成标准
- 能稳定拉取指定股票历史行情
- 能获取股票基础信息
- 能获取公告索引
- 数据可标准化返回
- provider 失败不导致系统整体崩溃

## Phase 2：技术分析 MVP

### 目标
建立第一版可用于策略与研究的技术分析底层。

### 范围
- 常用指标
- 趋势评分
- 波动评分
- 支撑/压力位识别
- 基础形态检测

### 关键能力
- MA 系统
- MACD
- RSI
- ATR
- Bollinger Bands
- 成交量均线
- 趋势状态识别
- 关键价位识别

### 完成标准
- 对任意股票可生成完整技术特征
- 指标与评分结果稳定
- 单元测试覆盖关键计算逻辑

## Phase 3：单票研究 MVP

### 目标
实现“输入股票代码，输出研究与策略”。

### 范围
- 技术研究
- 基本面摘要
- 公告/新闻研究
- 结构化研究报告
- 结构化策略计划

### 关键能力
- 生成技术结论
- 生成事件摘要
- 生成基本面摘要
- 输出最终 action
- 输出 entry/exit/hold 条件

### 完成标准
- 输入股票代码即可得到完整研究结果
- 输出符合 schema
- 可用于后续保存和复盘

## Phase 4：选股器 MVP

### 目标
实现全市场选股能力。

### 范围
- 股票池加载
- 初筛规则
- 候选评分
- Top 股票深度研究
- 选股结果展示

### 关键能力
- 扫描全市场
- 过滤不合格股票
- 计算候选分数
- 输出候选名单
- 生成候选研究摘要

### 完成标准
- 可生成“可买 / 观察 / 回避”名单
- 每只股票有短理由
- 支持后续前端展示

## Phase 5：交易记录与复盘 MVP

### 目标
建立交易闭环能力。

### 范围
- 手工录入交易
- 存储当时决策快照
- 从交易生成复盘草稿
- 复盘结果编辑与回看

### 完成标准
- 用户可录入交易
- 可查看历史记录
- 可关联当时决策快照（含质量摘要与置信度）
- 可从交易生成复盘草稿并编辑 outcome/lesson tags

## Phase 6：轻前端完善

### 目标
构建轻量且可用的 Web 工作台。

### 范围
- 首页
- 选股页
- 单票研究页
- 交易记录页（可用）
- 复盘页（可用）

### 完成标准
- 用户不需要直接调用 API 也能使用系统
- 页面简单、清晰、可用
- 不做复杂交互，不做重设计系统

## Phase 7：策略优化与学习增强

### 目标
提升系统长期价值与可迭代能力。

### 范围
- 统计有效策略
- 识别无效 setup
- 自动归类交易模式
- 形成迭代建议

### 可选增强
- 更高级策略模板
- watchlist 管理
- 模拟交易
- 更精细的市场 regime 分类
- agent 化改造

### 完成标准
- 能从历史研究与交易中提取规律
- 能输出面向下一阶段优化的建议

## 版本规划

- v0.1：项目骨架 + 数据接入 + 技术分析底层
- v0.2：单票研究 + 结构化策略输出
- v0.3：全市场选股器
- v0.4：交易记录与复盘
- v0.5：轻量前端整合
- v0.6+：学习增强 / 更完整 agent 化 / 模拟交易 / 持仓管理

## 决策原则

当“快做”和“做稳”冲突时，优先选择：
1. 架构清晰
2. 数据稳定
3. 输出结构化
4. 易测
5. 易扩展

不优先选择：
- 一次性堆很多功能
- 看起来智能但不可维护
- 强依赖脆弱网页源
- 过早优化 UI
