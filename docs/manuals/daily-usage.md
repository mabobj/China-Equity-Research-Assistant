# 日常使用说明

这份文档按“先结论、再证据、后细节”的顺序，说明当前系统怎么用，以及它和长期方向之间的关系。

## 1. 推荐使用顺序

当前最推荐的日常使用顺序是：

1. 先在 `/screener` 看全市场候选
2. 再进入 `/stocks/[symbol]` 看单票工作台
3. 需要执行时保存判断、记录交易
4. 之后到 `/reviews` 生成和补全复盘

这条链路是当前产品主线。长期方向中的因子、预测、评估，会逐步继续服务这条链路，而不是替代它。

## 2. 单票工作台怎么用

页面：

- `/stocks/[symbol]`

建议顺序：

1. 先看“决策简报”
2. 再看“证据与风险”
3. 再看“行动建议”
4. 最后看详细模块

你在单票页会看到的核心模块包括：

- `DecisionBrief`
- `review-report v2`
- `debate-review`
- `strategy plan`
- `trigger snapshot`
- `predictive snapshot`

### 关于“研究结论”和“预测结论”

当前单票页同时可能出现两类信息：

1. 研究解释链  
   来自 `review / debate / strategy / decision brief`

2. 预测评估链  
   来自 `predictive_snapshot`

前者回答“为什么”，后者回答“从当前最小预测底座看，信号强度和置信度如何”。

这两者现在是并行对照关系，不应互相混淆。

## 3. 选股工作台怎么用

页面：

- `/screener`

当前主流程已经切换到 workflow 模式，不再以同步长请求为主。

操作方式：

1. 在“运行初筛工作流”里填写 `batch_size`
2. 提交后拿到 `run_id`
3. 页面轮询 workflow 运行状态
4. 查看最新批次摘要和结果表
5. 点击股票查看详情，再进入单票页

### 当前展示窗口是什么意思

当前结果展示按 `Asia/Shanghai` 的 17:00 窗口划分：

- 17:00 前：显示前一日 17:00 之后到当日 17:00 之前完成的结果
- 17:00 后：显示当日 17:00 之后到当前时刻完成的结果

### 结果里常见字段是什么意思

- `行情质量`：bars 清洗后的质量状态
- `财务质量`：financial_summary 清洗后的质量状态
- `公告质量`：announcements 清洗后的质量状态
- `质量折损`：质量门控对候选排序或分桶造成的影响
- `预测分`：当前最小预测主线给出的辅助分数
- `预测置信度`：预测结果可信度等级

这些字段的作用是帮助你理解“当前结果值不值得继续看”，而不是直接充当交易指令。

## 4. `review-report v2` 和 `debate-review` 分别是什么

### `review-report v2`

这是当前单票主研究产物。

它更像“结构化研究摘要”，用于回答：

- 为什么值得关注
- 主要优势和风险是什么
- 当前倾向的研究动作是什么

### `debate-review`

这是结构化裁决层。

它用于把多空视角、不同研究维度的分歧收敛成更清楚的裁决表达。

它不是自动交易引擎，也不应单独作为唯一买卖依据。

## 5. workflow 怎么用

当前工作流主入口：

- `POST /workflows/screener/run`
- `POST /workflows/deep-review/run`
- `GET /workflows/runs/{run_id}`

前端已经把 workflow 主路径包装到页面里，一般不需要你手动调接口。

你需要理解的是：

- workflow 用来处理长任务
- 页面会先拿 `run_id`
- 再轮询运行状态与结果摘要
- 这比同步长请求更稳定

## 6. 怎么查看 workflow 运行记录

你可以通过两种方式看：

1. 前端页面内的 workflow 结果面板
2. 后端接口：
   - `GET /workflows/runs/{run_id}`

如果是后端排查，也可以看本地运行记录和日志。

## 7. 当前规则版与 LLM 版的区别

### 规则版

适合：

- 日常主链路
- 可重复、可追溯、可测试场景

特点：

- 稳定
- 可复现
- 更适合作为当前基础决策链

### LLM 版

适合：

- 补充解释
- 文本归纳
- 多视角表达

特点：

- 更强的表达能力
- 失败时会自动回退到规则版
- 不替代确定性计算

## 8. 交易记录与复盘怎么用

### 保存判断

在单票页点击“保存本次判断”，系统会固化一份 `decision snapshot`。

### 记录交易

你可以在：

- 单票页
- `/trades`

创建交易记录。

系统会关联当时的：

- 决策动作
- 置信度
- 数据质量摘要
- 预测元数据

### 生成复盘

在 `/reviews` 中从交易生成复盘草稿，查看：

- 原判断快照
- 执行路径
- 复盘结论
- lesson tags

## 9. 当前系统与长期因子主线的关系

当前你在页面上使用的是“产品工作台层”。

长期方向里的：

- 点时特征
- 标签
- 回测
- 预测
- 因子验证

不会脱离这些页面单独存在，而是逐步成为这些页面背后的能力底座。

所以现在的正确用法不是“等未来全部做完再用”，而是：

- 先把当前工作台用起来
- 再让后续能力逐步增强它的判断质量和可验证性
