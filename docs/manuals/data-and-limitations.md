# 数据源与边界说明

这份文档回答两个问题：

1. 当前系统的数据从哪里来
2. 当前系统适合做什么，不适合做什么

## 1. 当前数据源

当前项目已经接入或预留了以下数据源能力：

- AKShare
- Baostock
- CNINFO
- Eastmoney
- mootdx（本地验证版）

这些数据源并不一定在任意机器、任意时刻都全部可用，因此系统设计上默认允许 provider 失效、返回空结果或回退。

## 2. mootdx 当前支持范围

`mootdx` 当前是“本地行情验证版”。

你可以把它理解成：

- 依赖本机通达信目录
- 用于补充或验证本地可用的数据能力
- 更适合作为 provider registry 中的一个可选能力来源

当前不应该假定：

- 任意 symbol 都一定有 mootdx 数据
- 任意时间粒度都一定有 mootdx 支持
- 打开 `ENABLE_MOOTDX=true` 后就一定优先成功

如果 `MOOTDX_TDX_DIR` 不正确，或者本地目录本身不完整，provider 可能会直接不可用。

## 3. 当前不支持哪些能力

当前阶段明确不支持：

- 自动实盘交易
- 券商下单
- 高频交易
- 实时盘中自动执行
- 持仓管理
- 自动复盘
- 调度器 / 队列 / DAG 平台

前端里的 `/trades` 和 `/reviews` 目前也只是占位页，不代表这些业务已经完成。

## 4. factor / review / debate / strategy 分别是什么

## factor

作用：

- 计算结构化因子
- 输出 `alpha / trigger / risk`
- 提供因子组摘要

适合：

- 做快速横向比较
- 为单票研究和选股提供底层分数

不适合：

- 单独作为最终买卖依据

## review

作用：

- 把单票研究拆成技术、基本面、事件、情绪四个维度
- 给出结构化的最终判断

适合：

- 快速读懂一只股票“为什么值得关注或回避”

不适合：

- 替代完整的人工深度研究

## debate

作用：

- 通过固定角色结构化裁决
- 形成多空观点与首席裁决

适合：

- 辅助理解当前多空分歧
- 给 review 结果增加“可讨论”的结构

不适合：

- 把自由文本当成唯一交易依据

## strategy

作用：

- 把研究结论压缩成执行计划
- 输出入场、止损、止盈、持有、卖出规则

适合：

- 把研究结果转成结构化执行框架

不适合：

- 替代盘中实时人工判断

## 5. 系统当前适合什么

当前系统适合：

- 做 A 股单票研究
- 做结构化初筛和深筛
- 对研究结果进行可回看的裁决和策略收敛
- 在本地持续迭代研究流程

## 6. 系统当前不适合什么

当前系统不适合：

- 把结果直接当成实盘自动执行信号
- 假定所有数据都实时、完整、无延迟
- 假定 LLM 结果一定比规则版更可靠
- 忽略 provider 空结果、接口波动和本地环境差异

## 7. 结果边界

必须明确：

- review / debate / strategy 是研究辅助，不是收益承诺
- provider 结果可能存在延迟、缺字段或短时不可用
- LLM 版是增强层，不是确定性计算层
- workflow 是编排层，不是新的研究算法层

## 8. 关于 LLM 版的边界

LLM 当前负责：

- 财报 / 公告 / 研究结论的结构化解释
- 多信息源的文字综合
- 角色化视角表达

LLM 当前不负责：

- 技术指标计算
- 因子分数计算
- 风险边界计算
- 价格区间判定

如果 LLM 输出不合格，系统会自动回退到规则版。

## 9. 使用免责声明

本系统的输出只适合作为研究与决策辅助材料，不构成任何投资建议或收益保证。

在任何真实交易动作前，你都应该至少再确认：

- 数据日期是否最新
- 当前 provider 是否稳定
- 风险边界是否可接受
- 自己是否理解 review / debate / strategy 的含义
