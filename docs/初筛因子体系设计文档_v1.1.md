# 初筛因子体系设计文档 v1.1

## 1. 文档目标

本文档用于定义 A 股项目中“初筛”模块的因子体系、数据分层、存储策略、计算口径与实现边界。

目标不是直接预测“明天一定上涨的股票”，而是：

- 用最少、最稳、最便宜的数据
- 在全市场范围内稳定运行
- 每天盘后稳定产出一批“值得继续深度筛选”的股票
- 为后续深度筛选、研究报告、交易策略提供高质量候选池
- 为后续回测、复盘、因子迭代提供可追溯的日度因子快照

---

## 2. 初筛的产品定位

### 2.1 初筛解决什么问题

初筛只解决一个问题：

**从全市场中，找出当前结构健康、可交易、值得继续看的股票。**

它不直接解决以下问题：

- 不直接给最终买卖结论
- 不直接输出完整研究报告
- 不直接输出完整交易策略
- 不直接做财务深度判断
- 不直接做公告/新闻深度解读

### 2.2 初筛与深度筛选的分工

#### 初筛
负责：
- 压缩股票池
- 过滤不可交易或无意义样本
- 找出趋势/结构/活跃度较好的候选
- 为深度筛选提供高召回候选池

#### 深度筛选
负责：
- 财务质量判断
- 公告/事件判断
- 板块与行业相对强弱判断
- 交易计划生成
- 风险点与失效条件判断

### 2.3 初筛的设计原则

1. 因子原子化
2. 依赖低、稳定性高
3. 支持全市场批量运行
4. 高召回，低误杀
5. 输出可解释
6. 运行结果可保存、可追溯、可复盘

---

## 3. 总体分层设计

初筛链路拆成五层：

```text
原始数据
  -> 过程数据
  -> 原子因子
  -> 横截面因子 / 连续性因子
  -> 复合因子 / 规则结果
  -> 初筛名单
```

### 3.1 原始数据
直接来自 provider 的最基础数据。

### 3.2 过程数据
由原始数据计算得出的中间指标，不直接作为最终判断结果，但为因子提供输入。

### 3.3 原子因子
每个因子只表达一个清晰事实。

### 3.4 横截面因子
同一个交易日，在全市场或行业内比较不同股票的相对位置。

### 3.5 连续性因子
同一只股票某个因子在多日维度上的持续性、改善性、压缩性或积累性。

### 3.6 复合因子
由多个原子因子、横截面因子、连续性因子组合，用于表达更高层含义。

### 3.7 初筛结果
根据硬过滤、原子因子、复合因子组合后输出：

- BUY_CANDIDATE
- WATCHLIST
- AVOID

## 5. 过程数据设计与计算定义

### 5.0 统一计算约定

本文档默认采用以下统一口径：

- `t` 表示当前交易日
- `close_t` 表示第 `t` 日收盘价
- `open_t`、`high_t`、`low_t`、`volume_t`、`amount_t` 同理
- 所有滚动窗口默认包含当日
- 第一版建议全部使用后复权日线
- 第一版过程数据优先面向日频初筛，不考虑盘中级别

---

## 5.1 趋势类过程数据

### ma5 / ma10 / ma20 / ma60 / ma120

定义：

`ma_n(t) = mean(close_{t-n+1}, ..., close_t)`

例如：

`ma20(t) = mean(close_{t-19}, ..., close_t)`

用途：
- 判断价格位置
- 判断均线排列
- 计算趋势类原子因子

### ma20_slope

建议第一版定义为：

`ma20_slope(t) = ma20(t) / ma20(t-5) - 1`

解释：
- 过去 5 个交易日，MA20 是否向上抬升
- 正值表示中短期趋势线向上

### ma60_slope

建议第一版定义为：

`ma60_slope(t) = ma60(t) / ma60(t-10) - 1`

解释：
- 过去 10 个交易日，MA60 是否向上抬升
- 正值表示中期趋势线向上

### close_percentile_60d

定义为当前收盘价在过去 60 日区间中的位置：

`close_percentile_60d(t) = (close_t - rolling_low_60(t)) / (rolling_high_60(t) - rolling_low_60(t))`

其中：

- `rolling_high_60(t) = max(high_{t-59}, ..., high_t)`
- `rolling_low_60(t) = min(low_{t-59}, ..., low_t)`

结果范围一般在 `[0, 1]`。

### return_20d

定义：

`return_20d(t) = close_t / close_{t-20} - 1`

### return_60d

定义：

`return_60d(t) = close_t / close_{t-60} - 1`

---

## 5.2 波动类过程数据

### atr20

先定义单日真实波幅 `TR_t`：

`TR_t = max(high_t - low_t, abs(high_t - close_{t-1}), abs(low_t - close_{t-1}))`

然后：

`atr20(t) = mean(TR_{t-19}, ..., TR_t)`

第一版使用简单移动平均即可，不必先使用 Wilder smoothing。

### atr20_pct

定义：

`atr20_pct(t) = atr20(t) / close_t`

用途：
- 用相对波动率比较不同价位股票
- 更适合横向比较

### range_20d

定义为过去 20 日最高最低的相对区间：

`range_20d(t) = rolling_high_20(t) / rolling_low_20(t) - 1`

其中：

- `rolling_high_20(t) = max(high_{t-19}, ..., high_t)`
- `rolling_low_20(t) = min(low_{t-19}, ..., low_t)`

### volatility_20d

先定义单日对数收益率：

`r_t = ln(close_t / close_{t-1})`

然后定义：

`volatility_20d(t) = std(r_{t-19}, ..., r_t)`

第一版建议直接使用 20 日收益率标准差，不强制年化。

---

## 5.3 流动性类过程数据

### volume_avg_5d

`volume_avg_5d(t) = mean(volume_{t-4}, ..., volume_t)`

### volume_avg_20d

`volume_avg_20d(t) = mean(volume_{t-19}, ..., volume_t)`

### amount_avg_5d

`amount_avg_5d(t) = mean(amount_{t-4}, ..., amount_t)`

### amount_avg_20d

`amount_avg_20d(t) = mean(amount_{t-19}, ..., amount_t)`

### volume_ratio_1d_20d

定义：

`volume_ratio_1d_20d(t) = volume_t / volume_avg_20d(t)`

解释：
- 大于 1 表示相对放量
- 小于 1 表示相对缩量

### amount_ratio_1d_20d

定义：

`amount_ratio_1d_20d(t) = amount_t / amount_avg_20d(t)`

---

## 5.4 结构位置类过程数据

### rolling_high_20

`rolling_high_20(t) = max(high_{t-19}, ..., high_t)`

### rolling_low_20

`rolling_low_20(t) = min(low_{t-19}, ..., low_t)`

### rolling_high_60

`rolling_high_60(t) = max(high_{t-59}, ..., high_t)`

### rolling_low_60

`rolling_low_60(t) = min(low_{t-59}, ..., low_t)`

### support_level

第一版建议采用简单稳定定义：

`support_level(t) = rolling_low_20(t)`

后续 v2 可升级为：
- swing low
- 均线支撑
- 平台支撑
- 多支撑融合

### resistance_level

第一版建议采用简单稳定定义：

`resistance_level(t) = rolling_high_20(t)`

后续 v2 可升级为：
- swing high
- 平台高点
- 结构性压力位

### distance_to_support_pct

定义：

`distance_to_support_pct(t) = (close_t - support_level(t)) / close_t`

值越小，说明越贴近支撑。

### distance_to_resistance_pct

定义：

`distance_to_resistance_pct(t) = (resistance_level(t) - close_t) / close_t`

值越小，说明越贴近压力位。

---

## 5.5 横截面过程数据（原 v1 中“切面过程数据”）

说明：
- 本文档将原“切面过程数据”统一改名为“横截面过程数据”
- 它表示同一天内，不同股票之间的相对位置

### amount_rank_pct

定义：
- 某一天全市场可筛选股票中，该股票 `amount_avg_20d` 所处的分位数

设当天合格股票池大小为 `N_t`，股票 i 的排名为 `rank_i`，则：

`amount_rank_pct(i,t) = (rank_i - 1) / (N_t - 1)`

范围 `[0, 1]`。

### return_20d_rank_pct

定义：
- 当天全市场股票中，该股票 `return_20d` 所处的分位数

`return_20d_rank_pct(i,t) = (rank_i - 1) / (N_t - 1)`

### trend_score_raw

建议第一版先定义一个基础原始趋势分：

`trend_score_raw = 20 * I(close > ma20)
                 + 20 * I(close > ma60)
                 + 15 * I(ma5 > ma10 > ma20)
                 + 15 * I(ma10 > ma20 > ma60)
                 + 10 * I(ma20_slope > 0)
                 + 10 * I(ma60_slope > 0)
                 + 10 * close_percentile_60d`

其中 `I(condition)` 为布尔条件成立时取 1，否则取 0。

### trend_score_rank_pct

定义：
- 当天全市场股票中，该股票 `trend_score_raw` 所处的分位数

`trend_score_rank_pct(i,t) = (rank_i - 1) / (N_t - 1)`

### atr_pct_rank_pct

定义：
- 当天全市场股票中，该股票 `atr20_pct` 所处的分位数

`atr_pct_rank_pct(i,t) = (rank_i - 1) / (N_t - 1)`

注意：
- 值高仅代表波动更大，不代表更好

### industry_relative_strength_rank_pct

建议定义为：

第一步：计算个股 20 日收益率 `return_20d(i,t)`

第二步：计算所属行业 g 在当天的收益率中位数：

`industry_median_return_20d(g,t)`

第三步：计算行业内相对强弱：

`industry_relative_strength(i,t) = return_20d(i,t) - industry_median_return_20d(g,t)`

第四步：在行业内排名并转成分位数：

`industry_relative_strength_rank_pct(i,t) = (rank_i - 1) / (N_g - 1)`

其中 `N_g` 为该行业当日样本数。

## 6. 因子分类与术语修正

为避免歧义，建议采用如下术语：

### 6.1 原子因子
直接表达一个清晰事实。

例如：
- `price_above_ma20`
- `ma20_slope_positive`
- `liquidity_ok`

### 6.2 横截面因子
同一天内，把一只股票放到全市场或行业中比较其相对位置。

例如：
- `amount_rank_pct`
- `trend_score_rank_pct`
- `industry_relative_strength_rank_pct`

### 6.3 时间序列因子
同一只股票，某个因子在多日维度上的变化轨迹。

例如：
- 最近 5 天 `trend_score_raw` 的变化
- 最近 3 天 `amount_rank_pct` 的变化

### 6.4 连续性因子
由时间序列因子进一步加工而成，用于表达持续性、改善性、压缩性、积累性。

例如：
- `trend_rank_persistence_5d`
- `amount_rank_improving_3d`
- `volatility_compression_streak`

### 6.5 复合因子
由多个原子因子、横截面因子、连续性因子组合而成的高层含义因子。

例如：
- `trend_quality_score`
- `setup_readiness_score`

- bool

- 名称中不包含 ST / *ST

类型：
- bool

- 上市交易日数量 >= 最低阈值，例如 60 或 120 个交易日

类型：
- bool

### 8.4 price_range_ok
定义：
- 最新收盘价在允许区间内

类型：
- bool

### 8.5 liquidity_ok
定义：
- 20 日平均成交额达到最低要求

类型：
- bool

### 8.6 data_integrity_ok

类型：

## 10. 位置与形态原子因子

### 10.1 near_breakout
建议定义：
- `distance_to_resistance_pct(t) <= breakout_threshold`
- 例如阈值可先设为 0.03

类型：
- bool

### 10.2 pullback_to_ma20
建议定义：
- 整体趋势向上
- 且 `abs(close_t / ma20(t) - 1) <= pullback_band`

类型：
- bool

### 10.3 pullback_to_ma10
建议定义：
- 整体趋势向上
- 且 `abs(close_t / ma10(t) - 1) <= pullback_band`

类型：
- bool

### 10.4 close_near_support
建议定义：
- `distance_to_support_pct(t) <= support_near_threshold`
- 例如阈值 0.03

类型：
- bool

### 10.5 not_too_close_to_resistance
建议定义：
- `distance_to_resistance_pct(t) >= min_room_threshold`
- 例如阈值 0.02 或 0.03

类型：
- bool

### 10.6 structure_not_broken
建议定义：
- `close_t >= ma20(t)` 或未明显跌破 `support_level`
- 且最近 N 日无明显结构破坏信号

类型：
- bool

## 12. 横截面因子与连续性因子

### 12.1 是否需要保存横截面因子

答案：需要。

原因：

1. 横截面因子描述的是“相对市场位置”
2. 适合全市场排序
3. 能提升初筛的稳定性和可比较性
4. 能进一步派生出连续性因子

### 12.2 推荐保存的日度横截面因子

- amount_rank_pct
- return_20d_rank_pct
- trend_score_rank_pct
- atr_pct_rank_pct
- industry_relative_strength_rank_pct

### 12.3 是否需要保存每日因子结果

答案：需要，而且最好不是只保存“入选股票”，而是保存“全量可评估股票”的日度因子快照。

原因：

1. 便于分析入池/出池原因
2. 便于研究因子变化轨迹
3. 便于回测、复盘、版本比较
4. 便于生成连续性因子
5. 存储成本相对可控

### 12.4 多日连续结果是否可作为因子

答案：可以，而且很有价值。

连续性因子本质上表达的是：
- 因子是否持续成立
- 因子是否持续改善
- 因子是否持续收缩/积累

它们往往比单日信号更稳定。

### 12.5 推荐的连续性因子

#### trend_rank_persistence_5d
定义：
- 过去 5 天 `trend_score_rank_pct` 连续位于前 20%

#### amount_rank_improving_3d
定义：
- 过去 3 天 `amount_rank_pct` 连续改善

#### rs_persistence_5d
定义：
- 过去 5 天 `industry_relative_strength_rank_pct` 持续位于较高水平

#### volatility_compression_streak
定义：
- 波动分位数连续多日收缩

#### breakout_pressure_building
定义：
- 多日接近压力位但未明显回落

## 13. 连续性因子计算建议

### trend_rank_persistence_5d

定义：

过去 5 天中，若 `trend_score_rank_pct >= 0.8` 连续成立，则记为真。

实现建议：
- 可存为 bool
- 也可存为 count 型：
  - `trend_rank_top20_days_5d`

### amount_rank_improving_3d

定义：

若满足：

- `amount_rank_pct(t-2) < amount_rank_pct(t-1) < amount_rank_pct(t)`

则为真。

### rs_persistence_5d

定义：

过去 5 天中，`industry_relative_strength_rank_pct >= threshold` 的天数达到设定值。

例如：
- 阈值 0.7
- 过去 5 天至少有 4 天成立

### volatility_compression_streak

定义：

记录波动分位数连续下降或处于低位区间的连续天数。

例如：
- `atr_pct_rank_pct <= 0.4`
- 连续满足的天数即为 streak

### breakout_pressure_building

定义：

过去 N 天中：
- `distance_to_resistance_pct` 持续较小
- 且回撤不深
- 且结构未破坏

这类因子适合用于识别“逼近突破位”的压缩状态。

---

## 14. 复合因子设计

复合因子由原子因子、横截面因子、连续性因子组合而成，不直接从原始数据计算。

### 14.1 tradability_score
由以下因子组合：
- is_tradeable
- is_non_st

## 15. 初筛结果生成逻辑

### 15.1 结果类别

- BUY_CANDIDATE
- WATCHLIST
- AVOID

### 15.2 推荐判定逻辑

#### AVOID
满足以下任一情况：
- 硬过滤因子不通过
- 数据完整性不足
- 流动性不够
- 波动过高且结构破坏

#### WATCHLIST
满足：
- 可交易
- 趋势质量中等及以上
- 位置尚可，但未到最好时点
- 或者趋势良好但接近压力位等待确认

#### BUY_CANDIDATE
满足：
- 可交易
- 趋势质量较高
- 位置合理（回踩或接近突破）
- 风险可控
- 参与度较好

### 15.3 初筛不是最终买点判断

BUY_CANDIDATE 的含义不是“直接买入”，而是：
- 值得进入深度筛选
- 值得进入次级研究链路
- 值得生成进一步策略计划

## 18. 哪些内容不进入初筛

以下内容一律不作为初筛硬依赖：

- 财务摘要
- 公告列表
- 公告正文
- 新闻与情绪
- LLM 研究结果

这些内容留给深度筛选阶段。

## 19. 初筛运行方式建议

初筛必须改为“盘后预计算”模式。

### 推荐链路

```text
每日盘后
  -> 更新 symbol_master_daily
  -> 更新 bars_daily_compact
  -> 生成 screener_process_daily
  -> 生成 screener_atomic_factors_daily
  -> 生成 screener_cross_section_daily
  -> 生成 screener_temporal_factors_daily
  -> 生成 screener_composite_factors_daily
  -> 生成 screener_candidates_daily
  -> 前端/API 只读结果
```

### 不推荐链路

```text
用户点击初筛
  -> 运行时逐只拉数据
  -> 逐只算指标
  -> 逐只算因子
  -> 实时生成结果
```

原因：
- 慢
- 易超时
- 易重复请求
- 易被限流
- 结果不稳定

## 20. 实现优先级建议

### Phase 1：最小可用初筛
只做：
- 股票主数据
- 压缩日线
- 硬过滤因子
- 趋势结构因子
- 基础输出列表

### Phase 2：增强版初筛
加入：
- 支撑/压力
- 活跃度因子
- 横截面因子
- 连续性因子

### Phase 3：与深筛衔接
加入：
- top candidates 推送深筛
- 初筛结果绑定 deep review 工作流
- 初筛与深筛联动展示

---

## 21. 成功标准

一个“有实际价值、可实际产出”的初筛产品，至少应满足：

1. 过去 20 个交易日中大多数交易日都能成功产出
2. 每日盘后在固定时间内完成
3. 结果结构稳定、可保存、可复用
4. 不依赖财务/公告/LLM 才能运行
5. 每只候选都有明确的固定原因
6. 可以直接驱动后续深筛
7. 能追溯任意一只股票某日的因子状态与入池/出池原因

---

## 22. 总结

初筛最重要的不是“聪明”，而是：

- 稳
- 快
- 省
- 可解释
- 可持续产出

本设计文档建议采用：

**主数据 + 日线 + 过程数据 + 原子因子 + 横截面因子 + 连续性因子 + 复合因子 + 固定规则输出**

来构建第一版初筛产品。

在这个框架下：

- 原子因子要尽量简单
- 复合因子要由原子因子组合
- 横截面因子要保存
- 连续性因子应当作为独立因子使用
- 每日因子结果应按天保存
- 财务、公告、新闻、LLM 等复杂输入后置到深度筛选阶段

这样才能把初筛真正做成一个有实际价值、可持续产出的产品。
