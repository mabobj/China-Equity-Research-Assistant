# 项目任务书（v2.1，产品化增强版）

本文档的角色已经收口为：

**记录当前产品化增强阶段的完成基线。**

它不再承担长期路线图全文的角色。长期方向请看：

- [系统架构](architecture.md)
- [路线图](roadmap.md)
- [当前阶段](current_phase.md)

## 1. 文档定位

`taskbook-v2.1` 记录的是这一轮产品化增强任务如何把项目从“工程控制台”推进到“可用工作台”。

它关注的是：

- 单票、选股、交易、复盘四条主交互链
- 预测与评估能力的最小接入
- 页面结构、术语、运行可见性、闭环可追溯性

它不替代长期北极星文档，也不替代当前阶段边界文档。

## 2. 当前状态

`v2.1` 定义的 6 个任务包已全部完成。

这意味着当前产品层已经完成：

- 单票工作台产品化
- 选股工作台产品化
- 交易与复盘最小闭环可用化
- 预测主线最小接入
- 预测结果、模型版本建议与 workflow 可见性接入

## 3. 六个任务包完成摘要

### 包 1：认知收口包

已完成：

- 清理旧叙事与占位文案
- 统一主入口说明
- 建立 v2.1 产品化增强任务书

### 包 2：单票体验重构包

已完成：

- 单票页以结论优先
- 运行详情下沉
- 保存判断与记录交易入口接入

### 包 3：选股体验重构包

已完成：

- 结果优先展示
- workflow 驱动而非同步长请求
- 批次结果台账与详情查看

### 包 4：交易复盘体验收口包

已完成：

- 交易记录最小可用
- 复盘草稿生成与编辑
- 原判断快照 / 执行路径 / 复盘结论的对照视图

### 包 5：预测底座包

已完成：

- dataset / label / prediction / backtest / evaluation 最小主线
- 特征与标签落盘
- 回测与评估基础能力

### 包 6：预测接入包

已完成：

- `predictive_snapshot` 接入 `workspace-bundle`
- 选股、深筛、交易、复盘可查看预测元数据
- 模型版本建议接入单票页、选股页与 workflow 运行详情

## 4. v2.1 的真实产出

从代码和产品视角看，v2.1 的真实产出可以总结为三点：

1. 当前工作台壳层已经形成  
   用户可以完成“选股 -> 单票判断 -> 记录交易 -> 生成复盘”的最小闭环。

2. 数据、研究、策略、交易、复盘都具备了结构化输出  
   系统已经不是只会吐自由文本的 demo。

3. 预测与评估主线已经接进来  
   虽然还不是长期因子系统的终态，但已经不是空白骨架。

## 5. v2.1 与长期方向的关系

v2.1 不是长期终点，而是长期方向里的“壳层产品化阶段”。

它的意义在于：

- 保留现有工作台价值
- 为后续预测、评估、因子验证继续提供产品入口
- 避免团队继续在早期 MVP 叙事里来回摇摆
