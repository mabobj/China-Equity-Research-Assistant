# A 股因子字典 / 因子目录说明书

**A 股公开信息驱动多层多因子系统 - 第一版**

版本：v1.0  
日期：2026-04-09  
OpenAI / ChatGPT 协助整理

## 1. 文档目的

本说明书用于定义 A 股公开信息驱动多层多因子系统的第一版因子字典与因子目录标准，作为后续：

- 因子开发与实验的统一依据
- 因子命名、字段、口径与版本管理规范
- 因子研究、验证、上线与淘汰流程的参考文档
- 后续 Codex 开发任务与数据库/接口设计的上游输入

本版强调 **公开信息、结构化、可验证、可淘汰、可持续演化**，不以内幕、未公开重大信息或不可合规复现的信息作为因子来源。

## 2. 因子字典设计原则

### 2.1 基本原则

1. **因子必须可定义**：每个因子都要能写成明确的对象与规则，而不是模糊描述。
2. **因子必须可追溯**：必须能追溯数据源、清洗版本、抽取版本、公式版本和验证版本。
3. **因子必须可验证**：任何因子都必须经过样本外、成本后、状态分层验证。
4. **因子必须可淘汰**：衰减、拥挤、重复、失去可交易性的因子必须进入降权或退役流程。
5. **因子必须分层使用**：顶层状态、行业/板块、个股 alpha、风险惩罚不能混成同一层简单打分。

### 2.2 因子生命周期

- `candidate`：候选因子，只有定义，无正式验证结果
- `in_validation`：验证中，已有初步实验，但未完成样本外检验
- `approved`：验证通过，可进入组合候选池
- `live`：已上线观察或实盘 shadow 运行
- `decay`：表现衰减，进入降权 / 重新验证阶段
- `retired`：退役，不再进入新策略

### 2.3 因子统一字段规范

| 字段 | 说明 |
|---|---|
| factor_code | 因子唯一编码 |
| factor_name | 因子名称 |
| layer | 所属层级 |
| priority | 优先级，P0 为首批必做，P1 为增强项 |
| horizon | 主要作用持有期或观察期 |
| definition | 因子定义 |
| data_source | 主要数据源 |
| calculation_logic | 计算逻辑 |
| usage | 用途与进入哪一层决策 |
| validation_focus | 验证重点 |
| risk_notes | 使用风险与口径注意事项 |

## 4. 详细因子条目

### 4.1 市场趋势状态因子（REGIME_01）

- **层级**：顶层状态层
- **优先级**：P0
- **作用周期**：5-20个交易日

**定义**  
衡量全市场当前处于上升、震荡或下行状态的综合因子，用于决定趋势类与突破类因子的启用权重。

**数据源**  
上证指数、深成指、沪深300、中证1000、全A指数日线；可选涨跌家数、成交额结构。

**计算逻辑**  
综合5/10/20日收益、指数相对MA20/MA60位置、均线斜率、上涨家数占比，标准化后形成0-100分并映射为 up / neutral / down。

**用途**  
作为系统顶层 regime，不直接选股；控制趋势、突破、题材扩散类因子的整体权重和仓位上限。

**验证重点**  
分年份、分风格、分波动环境验证对下游因子收益分布的解释能力；关注 regime 切换边界是否稳定。

**使用注意事项**  
不得直接把指数短期涨跌当买卖信号；重点是给下层因子提供上下文。

### 4.2 市场波动状态因子（REGIME_02）

- **层级**：顶层状态层
- **优先级**：P0
- **作用周期**：3-10个交易日

**定义**  
衡量市场当前波动是低、正常还是高，用于区分“可持有的趋势”与“高噪音高冲击环境”。

**数据源**  
主要指数日线与分钟线、涨跌停家数、市场振幅、可选情绪代理。

**计算逻辑**  
使用10/20日实现波动率、指数振幅中位数、涨跌停比例、市场跳空率做标准化评分，输出 low / normal / high。

**用途**  
控制止损宽度、仓位、缩量回踩类因子的可用性；高波动状态下提高风险惩罚层权重。

**验证重点**  
验证其对回撤、滑点、策略盈亏比的解释能力；比较不同波动状态下同一因子的净值路径。

**使用注意事项**  
优先用于风控和调权，不建议作为直接 alpha。

### 4.3 外部风险冲击因子（REGIME_03）

- **层级**：顶层状态层
- **优先级**：P1
- **作用周期**：1-10个交易日

**定义**  
衡量海外风险和全球资产价格波动对A股的潜在冲击强度。

**数据源**  
USD/CNH、海外主要股指、原油、有色、黄金等公开价格；可选全球不确定性代理。

**计算逻辑**  
对美元、海外风险资产和大宗商品的短期异常波动做z-score聚合；结合A股开盘与隔夜外盘偏离形成外部冲击分数。

**用途**  
作为出口链、资源链、成长板块的条件变量；在高外部冲击时降低激进因子权重。

**验证重点**  
分行业检验，重点看对外需、资源、科技链的差异影响；避免把共性冲击误判为个股 alpha。

**使用注意事项**  
更适合做条件和惩罚器，不应直接给所有股票一刀切。

### 4.4 风格切换因子（大盘/小盘，成长/价值）（REGIME_04）

- **层级**：顶层状态层
- **优先级**：P1
- **作用周期**：5-20个交易日

**定义**  
识别当前市场偏大盘还是小盘、偏成长还是偏价值，以调节因子组合暴露。

**数据源**  
风格指数、代表性ETF、全市场分组收益。

**计算逻辑**  
计算大小盘相对收益、成长价值相对收益、风格成交额占比和持续性，形成风格偏好标签。

**用途**  
为事件、动量、质量、低波动等因子做风格条件化加权。

**验证重点**  
验证风格状态下的分层收益稳定性，以及对组合回撤和风格暴露的改善效果。

**使用注意事项**  
不追求精确预测风格转折，重点是避免错配。

### 4.5 行业相对强弱因子（IND_01）

- **层级**：行业/板块层
- **优先级**：P0
- **作用周期**：5-20个交易日

**定义**  
衡量某行业相对市场基准的超额强弱，用于区分行业顺风与逆风环境。

**数据源**  
申万/中信行业指数或自建行业收益序列；行业成交额与上涨家数。

**计算逻辑**  
计算行业5/10/20日超额收益、行业成交额占比变化、上涨家数占比，标准化后得到行业强弱分。

**用途**  
给行业顺风中的个股信号加分，逆风行业中的个股信号降权；用于选股分层。

**验证重点**  
分行业、分风格、分年份验证超额强弱的持续性和对个股 alpha 的放大效果。

**使用注意事项**  
该因子常是行业 beta，不要误当个股 alpha。

### 4.6 题材扩散率因子（IND_02）

- **层级**：行业/板块层
- **优先级**：P0
- **作用周期**：1-5个交易日

**定义**  
衡量某题材从龙头向板块成员扩散的速度和广度，判断题材处于早期、中期还是后期。

**数据源**  
题材板块成分股、日线/分钟线、涨停与大涨统计。

**计算逻辑**  
以龙头首次明确启动为事件原点，统计1/2/3日内同题材股票中上涨、放量、突破前高的比例，形成扩散率和扩散斜率。

**用途**  
用于消息市和题材市的阶段识别；帮助区分“可跟随扩散”与“尾端补涨”。

**验证重点**  
对不同题材类别、龙头强度和市场状态做分组，验证扩散率对后续题材收益和拥挤风险的解释能力。

**使用注意事项**  
必须先定义题材池与龙头识别规则，避免事后归因。

### 4.7 题材拥挤与分歧因子（IND_03）

- **层级**：行业/板块层
- **优先级**：P0
- **作用周期**：1-5个交易日

**定义**  
识别板块是在健康共振、强分歧博弈还是末端拥挤阶段。

**数据源**  
板块内个股收益、换手、振幅、可选社交热度。

**计算逻辑**  
用板块同步上涨率、换手中位数、龙头偏离度、收益分散度构造“拥挤/分歧”二维分数。

**用途**  
对追涨、题材扩散、突破类信号做风险惩罚或阶段性加权。

**验证重点**  
检验不同拥挤/分歧状态下，题材跟随策略的胜率、盈亏比和次日回撤差异。

**使用注意事项**  
拥挤高不一定马上结束，但会显著提高追高风险。

### 4.8 行业景气边际改善因子（IND_04）

- **层级**：行业/板块层
- **优先级**：P1
- **作用周期**：10-40个交易日

**定义**  
衡量一个行业内部盈利、订单、价格或政策景气是否在边际改善。

**数据源**  
行业内财务摘要、业绩预告、订单类公告、可选行业数据。

**计算逻辑**  
统计行业内利润增速改善比例、业绩预告超预期比例、订单公告密度变化，做行业内标准化评分。

**用途**  
为财务边际改善和事件因子提供行业景气上下文，减少个股信号孤立化。

**验证重点**  
按行业周期类型验证；重点看对中周期收益和持有期延展的解释能力。

**使用注意事项**  
第一版可用粗粒度替代，后续再做行业专有数据。

### 4.9 业绩预告意外度因子（EVT_01）

- **层级**：个股事件层
- **优先级**：P0
- **作用周期**：3-20个交易日

**定义**  
衡量业绩预告/快报对市场预期的偏离程度，是A股事件驱动体系的核心因子之一。

**数据源**  
巨潮/交易所公告、财务摘要、行业中位数对比。

**计算逻辑**  
用预告净利润区间中值、同比变化、是否修正预告、行业偏离度构造 surprise_score；负向预告给予非对称更高惩罚。

**用途**  
直接进入事件总分；驱动研究层的 event_score 和后续策略中的事件加权。

**验证重点**  
按行业、业绩预告类型、盈利区间大小做分层；重点看公告后1/3/5/10日收益和回撤表现。

**使用注意事项**  
必须严格时间对齐，盘后发布不能用于当天盘中决策。

### 4.10 回购 / 增持正向事件因子（EVT_02）

- **层级**：个股事件层
- **优先级**：P0
- **作用周期**：5-30个交易日

**定义**  
衡量回购、重要股东增持、员工持股等正向资本配置事件的力度。

**数据源**  
正式公告文本。

**计算逻辑**  
提取事件类型、计划金额/市值、执行进度、是否完成、是否连续发生，生成 positive_event_score。

**用途**  
提升中短期研究中的事件可信度，也可作为风险缓冲项。

**验证重点**  
区分计划公告与实施完成、金额大小和连续性，比较事件后收益与回撤路径。

**使用注意事项**  
计划不等于落地，执行进度必须进入模型。

### 4.11 减持 / 问询 / 立案风险事件因子（EVT_03）

- **层级**：个股事件层
- **优先级**：P0
- **作用周期**：1-30个交易日

**定义**  
统一量化减持计划、监管问询、立案调查、风险提示等负向披露对交易风险的影响。

**数据源**  
正式公告、监管文件。

**计算逻辑**  
对事件按严重度分级：问询 < 风险提示 < 立案 / 退市风险；结合是否连续发生与是否未消除不确定性做负向累积。

**用途**  
作为风险惩罚层的重要输入，必要时直接否决 BUY。

**验证重点**  
检验不同严重度事件对后续收益、回撤、波动和成交行为的影响差异。

**使用注意事项**  
与社交热度联动时更有效，但第一版应独立可用。

### 4.12 重大合同 / 中标强度因子（EVT_04）

- **层级**：个股事件层
- **优先级**：P1
- **作用周期**：5-40个交易日

**定义**  
衡量重大合同、订单、中标公告对经营预期的冲击强度。

**数据源**  
公告文本、历史营收、市值。

**计算逻辑**  
抽取金额、金额/市值、金额/营收、连续性、行业匹配度，形成 contract_strength_score。

**用途**  
适用于工程、设备、军工、能源链等订单驱动行业。

**验证重点**  
按行业类型和订单相对体量验证，关注公告后趋势持续性而非次日情绪。

**使用注意事项**  
需防止把小额订单当大利好。

### 4.13 披露密度异常因子（EVT_05）

- **层级**：个股事件层
- **优先级**：P1
- **作用周期**：1-10个交易日

**定义**  
衡量某股票在短时间内公告密度是否异常，异常密集往往代表信息流增加或不确定性提升。

**数据源**  
公告索引列表。

**计算逻辑**  
统计近3/5/10日公告数量相对过去60日均值的z-score；结合公告类型正负权重形成净披露密度分。

**用途**  
作为事件研究辅助因子；辅助判断消息面是否进入“密集驱动阶段”。

**验证重点**  
验证公告密度异常对波动、换手、题材扩散和单票收益的解释能力。

**使用注意事项**  
公告多不等于利好，必须和事件类型一起看。

### 4.14 趋势对齐评分因子（TEC_01）

- **层级**：个股技术层
- **优先级**：P0
- **作用周期**：5-20个交易日

**定义**  
综合均线位置、均线斜率、MACD、RSI 等，衡量单票当前是否处于健康上升趋势。

**数据源**  
日线OHLCV。

**计算逻辑**  
价格相对MA20/MA60位置、MA20/MA60斜率、MACD柱体方向、RSI区间共同打分，形成0-100的 trend_alignment_score。

**用途**  
作为研究和选股中的核心技术总分，是多个技术子因子的上层聚合。

**验证重点**  
检验对未来5/10日收益分层能力，以及与纯价格动量因子的相关性与增量。

**使用注意事项**  
不应与简单收益动量重复计算过多。

### 4.15 突破保持率因子（TEC_02）

- **层级**：个股技术层
- **优先级**：P0
- **作用周期**：1-10个交易日

**定义**  
衡量价格突破近期高点后，能否有效站稳而不是立刻回落。

**数据源**  
日线OHLCV、成交量。

**计算逻辑**  
当日收盘突破N日高点且量能放大后，统计未来1~3日是否仍站在突破位之上，并计算最大回撤与量能保持率。

**用途**  
比单纯 breakout 更适合实战，用来过滤假突破。

**验证重点**  
对不同N值、量能阈值、市场趋势状态做walk-forward，关注成本后收益与回撤。

**使用注意事项**  
第一版先作为规则化因子，后续可加入分钟级确认。

### 4.16 缩量回踩质量因子（TEC_03）

- **层级**：个股技术层
- **优先级**：P0
- **作用周期**：3-15个交易日

**定义**  
衡量上升趋势中的回踩是否健康，适合波段低吸与二次入场。

**数据源**  
日线OHLCV、均线。

**计算逻辑**  
先要求中期趋势向上，再判断价格回到MA10/MA20附近时是否缩量、未破关键支撑、K线实体收缩，形成 pullback_quality_score。

**用途**  
作为策略层 pullback 类型的核心驱动因子。

**验证重点**  
验证不同回踩深度、量缩幅度、支撑距离对后续反弹成功率的影响。

**使用注意事项**  
缩量回踩比追突破更稳，但更依赖大盘和行业环境。

### 4.17 波动收敛因子（TEC_04）

- **层级**：个股技术层
- **优先级**：P1
- **作用周期**：3-15个交易日

**定义**  
识别波动持续收敛、等待方向选择的股票。

**数据源**  
日线OHLCV、ATR、布林带宽度。

**计算逻辑**  
用ATR占比、近20日布林带宽度变化、振幅中位数变化构造 volatility_contraction_score。

**用途**  
为突破类和事件驱动类因子提供“压缩后释放”的技术背景。

**验证重点**  
验证与突破保持率的配合效果，以及单独使用时的误报率。

**使用注意事项**  
必须和方向信号搭配，单独使用噪声较大。

### 4.18 日内资金路径一致性因子（MIC_01）

- **层级**：个股微观结构层
- **优先级**：P1
- **作用周期**：1-3个交易日

**定义**  
衡量日内资金是全天一致推升，还是冲高回落、骗线出货。

**数据源**  
1分钟或5分钟K线、集合竞价数据。

**计算逻辑**  
整合集合竞价强度、开盘30分钟收益、午后回撤率、尾盘净推升，形成 intraday_path_score。

**用途**  
跟踪量化与短线资金的公开路径痕迹；适合短周期策略增强。

**验证重点**  
分市场状态验证对次日延续和回撤的解释力；加入交易成本后复核。

**使用注意事项**  
分钟数据维护成本高，建议第二阶段接入。

### 4.19 板块同步交易强度因子（MIC_02）

- **层级**：个股微观结构层
- **优先级**：P1
- **作用周期**：1-3个交易日

**定义**  
识别同板块股票是否出现同步性很强的交易路径，反映同质化资金共振。

**数据源**  
板块成分股分钟线、换手、成交额。

**计算逻辑**  
计算板块内分钟收益相关性、同步放量比例、同向脉冲频次，形成 board_sync_score。

**用途**  
识别“量化和题材资金一起动了”的阶段，提高对跟随型机会的判断力。

**验证重点**  
验证高同步性下板块延续、分歧和回撤的差异；重点关注尾端拥挤风险。

**使用注意事项**  
不是越同步越好，需和拥挤度联合解释。

### 4.20 财务边际改善因子（FUN_01）

- **层级**：个股财务层
- **优先级**：P0
- **作用周期**：10-60个交易日

**定义**  
关注财务指标的改善而非绝对好坏，是A股中周期最有用的财务类因子之一。

**数据源**  
财务摘要、季报、业绩预告。

**计算逻辑**  
综合营收同比、净利润同比、毛利率变化、经营现金流变化、ROE变化，行业内标准化后加总。

**用途**  
提升事件和技术因子的真实性，过滤纯情绪炒作。

**验证重点**  
按行业、大小盘、风格环境分层，验证对未来1-3个月收益和回撤的解释力。

**使用注意事项**  
更适合做增强项，不适合单独做短线触发。

### 4.21 现金流质量因子（FUN_02）

- **层级**：个股财务层
- **优先级**：P1
- **作用周期**：20-60个交易日

**定义**  
衡量利润是否真实转化为经营现金流，识别“利润好看但含金量差”的公司。

**数据源**  
现金流量表摘要、利润表摘要。

**计算逻辑**  
使用经营现金流/净利润、经营现金流同比、应收与存货变化等构建 cash_quality_score。

**用途**  
为研究层提供财务风险识别，提高持有期安全边际。

**验证重点**  
在高景气和低景气行业分别验证，关注其对暴雷和回撤的过滤效果。

**使用注意事项**  
适合做负面过滤，而不是短线加速器。

### 4.22 资产负债安全因子（FUN_03）

- **层级**：个股财务层
- **优先级**：P1
- **作用周期**：20-60个交易日

**定义**  
衡量资产负债表的稳健性，识别高杠杆和潜在财务脆弱性。

**数据源**  
财务摘要、资产负债表。

**计算逻辑**  
基于负债率、短债压力、现金短债比、净资产变化构建 balance_safety_score。

**用途**  
作为风险层增强项，避免把短期强势票配到财务脆弱标的上。

**验证重点**  
检验在市场下行和融资收缩环境中的过滤能力。

**使用注意事项**  
更偏防守属性。

### 4.23 注意力扩散冲击因子（SOC_01）

- **层级**：传播/社交层
- **优先级**：P1
- **作用周期**：1-5个交易日

**定义**  
衡量某股票或题材的公开讨论是否从“知道”进入“快速扩散”阶段。

**数据源**  
股吧、雪球、互动平台、公开问答平台。

**计算逻辑**  
计算提及量z-score、热度变化率、跨平台共振、传播半衰期，并用账号质量和内容可信度加权。

**用途**  
不作为主 alpha，而是用来确认事件扩散、题材传播和拥挤早期信号。

**验证重点**  
验证其对短期换手、波动、题材扩散率和次日收益分层的解释能力。

**使用注意事项**  
必须去水军、去噪声、严格时间对齐。

### 4.24 情绪分歧因子（SOC_02）

- **层级**：传播/社交层
- **优先级**：P1
- **作用周期**：1-3个交易日

**定义**  
衡量公开讨论中多空观点的分歧程度，分歧往往对应方向选择或大波动临近。

**数据源**  
公开社交文本与互动数据。

**计算逻辑**  
计算正负向文本比例、情绪方差、高互动内容之间的一致性，形成 sentiment_divergence_score。

**用途**  
辅助区分“健康扩散”与“高分歧博弈”；可做风险预警。

**验证重点**  
验证在题材启动、分歧转一致、极端一致三个阶段的表现差异。

**使用注意事项**  
分歧本身不是好坏，需要和趋势、事件联动解释。

### 4.25 拥挤预警因子（SOC_03）

- **层级**：传播/社交层
- **优先级**：P1
- **作用周期**：1-5个交易日

**定义**  
识别某股票或题材是否出现公开注意力过热、观点同质化和高拥挤风险。

**数据源**  
热度、互动、价格偏离、换手。

**计算逻辑**  
综合热度z-score、情绪一致性、价格偏离支撑程度、换手升温速度，形成 crowding_warning_score。

**用途**  
作为风险惩罚层重要输入，用来过滤追高和末端接力。

**验证重点**  
关注高拥挤样本的次日冲高回落率、回撤和盈亏比恶化情况。

**使用注意事项**  
拥挤高不等于立刻做空，但通常不适合激进加仓。

## 6. 因子验证框架

1. **定义正确性检查**：确认没有未来函数、时间错配、口径漂移。
2. **覆盖率与缺失检查**：确认因子不是被少量样本或极端值支配。
3. **排序能力验证**：IC、RankIC、分层收益、命中率。
4. **样本外验证**：训练 / 验证 / 测试切分，以及 Walk-forward。
5. **市场状态分层**：强趋势、震荡、退潮、高波动、低波动环境分别验证。
6. **成本后验证**：加入滑点、换手、T+1、涨跌停限制、成交额过滤。
7. **组合增量验证**：确认其对现有因子组合有增量，而不是重复暴露。
